Engineering Capstone Project (S.F.U.)

CART-FOLLOW-X1
For my engineering capstone project, I worked with four other senior engineering students to build and design an automated cart. Our motivation for this project is to improve convenience for cargo carrying and that it can eventually be used in many different applications.

The cart has two main functionalities: Follow-mode and Assist-mode. Assist-mode is when the user presses the "Assist" button on the handle of the cart, the cart will then be motorized allowing the user to easily move the cart with the assistance of the motor. Follow-mode is the main feature of this cart. The user simply needs to switch the operation mode (from the cart handle) and the cart will start following the user, allowing the user to operate the cart "hands-free". The follow mechanism is implemented using ultrasonic sensors on the cart. The sensors serve as the tracking mechanism as well as collision detection. The "brain" of the cart is the control system, it was implemented with three Raspberry Pi B+ units and a DC Motor Driver.

Our team was split into two groups, three of us (myself included) would be responsible for the Control System design, while the other two would be responsible for the Mechanical design. The control system was designed using Python, and the main functionalities include: user-tracking (distance calculation and turning distance calculation), collsion detection (collision distance calculation), motor control (motor speed calculation), and user interface buttons (on/off switch, mode switch, and LEDs).

In the end, we were able to complete a prototype and demo the product to our professors and instructors. I was really proud of the end result, as we used everything we learned in engineering from our first year to our last to complete this project. This project had a mix of electronics, mechanical design, and coding. In the end, the project was a success as we managed to achieve an excellent grade on the project. Embedded Systems C Programming (S.F.U.)

UART Performance Monitor
For my Real Time & Embedded Systems course, I worked in a team of three to design a VGA and UART driver to create a graphical performance monitor that will run on an embedded system (PetaLinux) using a FPGA board (Zedboard). The graphical console is capable of printing typed text, while also collecting and displaying live metrics about the system.

My task for this project was to develop a timer driver and timer interrupt service routine to implement a timer to collect basic statistics from the operation of the UART. I was also responsible for implementing a VGA driver to allocate the frame buffer in kernel-space and allow access to the frame buffer from user-space. The metrics and statistics that I implemented include: Time since boot up; character received; average read/write time per character; total time reading/writing characters; number of characters sent/received since startup; and total consol time. In the timer driver design, I implemented both hardware and software timers as our design measured timing for receiving and sending characters.

My group members were responsible for designing the framework for graphics support, and designing a VGA typing console. In the end, we combined our sections for the final product. Our design utilizes screen space by dividing it into two equal sections horizontally, the top supporting the typing console, and the bottom used for displaying the system metrics. This layout allows for text to be readable and avoids excessive wrapping. As seen in the picture below, the top section allows user to type up to 360 characters before the screen clears, and the bottom displays live statistical information.
